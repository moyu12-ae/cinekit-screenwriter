# 提示词迭代总结与 Skill 扩展路线图

## 五轮迭代做了什么

| 轮次 | 核心改动 | 关键发现 |
|------|---------|---------|
| V1 | 初始提示词，7大类规则（文体/意象/呈现/细节/句式/基调/省略） | 规则太多、太像教材。子Agent能理解但执行差。明喻5-6个/段。 |
| V2 | 加入正向示范（苏童手法：数字、走神、樱桃细节）；规则从禁止转向示范 | 正向示范比禁止性规则有效得多。叙述者年龄锚定后AI腔自然减少。 |
| V3 | 加入风格参照段落；细化句式呼吸规则 | 参照段落导致抄袭问题。部分Agent退步（明喻失控回潮）。 |
| V4 | 去掉参照段落；强化8条核心规则；限制明喻喻体领域 | 质量方差大——段落1很好但段落3退步。单靠规则不能稳定约束。 |
| V5 | **两段式写作（初稿+强制修改）**；修正「量化一切」问题 | 明喻稳定在1个/段；结尾干净；叙述声音一致。这是关键突破。 |

## 核心教训

1. **禁止不如示范**：告诉AI「不要怎么做」不如给它看「应该怎么做」的具体例子。
2. **年龄锚定文笔**：把叙述者的年龄、认知能力和语言习惯绑定后，「成人化叙述」「诗意总结」自然减少。
3. **强制修改是关键**：让AI先写再改，比让它一边写一边遵守规则有效得多。两段式流程是V5最大的突破。
4. **量化思维是AI的直觉**：AI默认会把一切变成数字。需要明确说「不要量化一切，感受优先」。
5. **喻体领域限制有用**：限制明喻喻体只能来自场景内部领域（身体/衣物/建筑/武器/木石），大幅减少了意象散漫。

## 扩展为 Skill 的路线图

当前提示词可以作为一个「写作过滤器」使用。扩展为完整 Skill 时，需要加入：

### 第一阶段：多作家适配
- 将苏童替换为变量。让用户指定目标作家+作品
- 自动提取目标作家的文笔特征
- 根据特征调整提示词中的具体参数

### 第二阶段：场景类型分类
- 不同场景类型（氛围开场/动作杀戮/情感对峙/独白反思）可能需要微调规则
- 建立场景类型→规则权重的映射

### 第三阶段：自动化迭代
- 自动选取测试段落
- 自动生成场景概要
- 自动对比评估
- 自动记录经验并改进提示词

### 第四阶段：质量控制工具
- 自动扫描AI输出中的：明喻数量、纠正句式、情绪词、诗意结尾
- 输出修改建议
