# Skill 扩展路线图

## 当前状态

V9 是一个**三Agent流水线提示词框架**。它依赖人工完成以下工作：
- 编写动作/对白骨架
- 设计分镜策略初稿
- 选择质感参考文段
- 手动串联三个Agent的执行顺序
- 人工判断是否需要打回→修改→重新提交

## 目标状态

一个完整的**写作 Skill**，包含：
- 自动化或半自动化的流水线调度
- 目标作家风格自动分析和参数提取
- 场景类型→策略卡的自动匹配
- 测试集和迭代优化的自动化

---

## 第一阶段：场景预处理自动化

**目标**：减少人工准备骨架和分镜策略的工作量。

**功能**：
1. **骨架提取器**：输入一段小说原文或场景描述，自动提取动作序列和对话列表，生成动作/对白骨架
2. **分镜策略生成器**：根据场景类型（开场氛围/动作冲突/情感对峙/告别/死亡等）自动推荐分镜序列和景别分配
3. **质感参考匹配**：根据目标作家和场景类型，从作家的代表作中自动检索最匹配的质感参考文段

**技术方案**：
- 骨架提取：基于 NLP 的动作识别 + 对话抽取
- 分镜生成：场景类型分类器 + 模板匹配
- 质感匹配：文本相似度检索

---

## 第二阶段：多作家适配

**目标**：框架不局限于苏童，可适配任何目标作家。

**功能**：
1. **作家风格画像**：输入作家的代表作文段，自动分析并存储风格参数
   - 句式节奏特征（长句/短句比例、逗号/句号分布）
   - 用词倾向（口语化/书面化程度、色彩词密度、触觉词密度）
   - 感官配比（视觉/听觉/触觉/嗅觉的分布）
   - 意象偏好（自然物/人造物/身体、美/丑/中性）
   - 叙述距离（近/中/远、冷/温/热）
2. **场景类型库**：建立场景类型到策略参数的映射数据库
   - 每种场景类型的典型时距分配方案
   - 每种场景类型的典型境界分配方案
   - 每种场景类型的典型空间策略

**实现方式**：
- 用户指定目标作家 + 代表作品 → Skill 自动提取风格参数 → 存入用户配置
- 用户指定场景类型 → Skill 自动加载对应的策略模板

## 第三阶段：全自动流水线

**目标**：用户只需提供场景概要，Skill 自动完成从策略到终版的全部流程。

**架构**：
```
用户输入：场景概要（一段自然语言描述）
        │
   ┌────▼──────────────────────────┐
   │  预处理器                      │
   │  ①提取动作/对白骨架             │
   │  ②场景类型分类                 │
   │  ③匹配分镜策略模板             │
   │  ④检索质感参考文段             │
   └────┬──────────────────────────┘
        │
   ┌────▼──────────────────────────┐
   │  Agent A：导演                 │
   │  [加载作家风格参数]             │
   │  ①叙事定位 ②时距分配 ③境界分配  │
   │  ④空间策略 ⑤质感分析 ⑥分镜脚本  │
   └────┬──────────────────────────┘
        │
   ┌────▼──────────────────────────┐
   │  Agent B：写手                 │
   │  第一人称描写层                 │
   └────┬──────────────────────────┘
        │
   ┌────▼──────────────────────────┐
   │  Agent C：编辑                 │
   │  抄袭检查 + 质量审核            │
   │  通过 → 输出                   │
   │  打回 → Agent B 修改（最多3次）  │
   └────┬──────────────────────────┘
        │
     最终正文
```

---

## 第四阶段：持续优化与社区协作

**目标**：让 Skill 在持续使用中自我改进。

**功能**：
1. **质量评分系统**：对每次输出进行自动评分（明喻控制/纠正句式/景别平衡/对话自然度等维度），用户也可手动打分
2. **策略参数调优**：根据评分反馈自动微调策略参数（如某场景类型的特写占比从40%调到35%效果更好）
3. **风格参数学习**：随着用户使用，不断细化作家风格画像
4. **社区策略库**：用户可分享自己调优的策略卡模板，形成社区知识库

## 优先级建议

| 阶段 | 功能 | 优先级 | 预计工作量 |
|------|------|--------|-----------|
| 0 | 将当前 V9 框架整理为结构化配置文件 | 🔴 立即 | 1-2天 |
| 1.1 | 场景类型→策略模板映射（手动编写6-8个常见场景类型） | 🔴 高 | 3-5天 |
| 1.2 | 骨架提取器（规则版，基于关键词匹配） | 🟡 中 | 3-5天 |
| 1.3 | 质感参考自动匹配（基于文本相似度） | 🟡 中 | 2-3天 |
| 2.1 | 作家风格画像模板（手动填写版） | 🔴 高 | 1-2天 |
| 2.2 | 作家风格自动分析器 | 🟢 低 | 5-7天 |
| 3.1 | 自动化流水线调度（串联三个Agent的执行） | 🟡 中 | 3-5天 |
| 3.2 | 自动打回重试循环（最多3次） | 🟡 中 | 2-3天 |
| 4.1 | 质量自动评分系统 | 🟢 低 | 5-7天 |
| 4.2 | 策略参数自动调优 | 🟢 低 | 7-10天 |
