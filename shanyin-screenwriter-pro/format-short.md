# 短片格式


## 核心特征

- **时长**：5-10分钟
- **角色**：1-2个主角，有完整的Want/Need/Arc
- **结构**：四段式（起因→发展→高潮→结局），聚焦单一核心事件
- **弧光**：一次完整的A→B变化
- **世界观**：简洁明了，30秒内观众理解规则
- **节奏**：信息密度高，每个场景都必须推进情节或揭示性格

## 短片的核心约束

短片只允许一个"单一核心事件"，不能有支线、不能分散焦点。
所有的场景都服务于同一个戏剧动作——主角追求目标、遭遇阻碍、做出选择。

除非用户特别要求，所有设定都应发生在现实主义背景之下，禁止过于夸张或过于戏剧性的设计。

## 完整八步工作流

### 第一步：破题与核心动作

**目标：** 确立短片的"核"。
**输入请求：** 如果用户已有构思，直接使用。如果用户没有明确想法，用 core-methodology.md 第九节的选题工具库通过自然对话引导（不要展示路径菜单）。

**执行逻辑：**
1. 基于用户输入，构思3-5个核心戏剧动作
2. 戏剧动作 = 目标 (Goal) + 阻碍 (Conflict)，必须是"有目的的追求"
3. 确保事件聚焦，适合短片体量

**输出格式（严格按此输出，不添加场景等未提及内容）：**

* **方案 A/B/C：**
  * 主角：
  * 目标 (Want)：
  * 阻碍 (Conflict)：
  * 一句话故事梗概 (Logline)：

### 第一步半：主题论点

**目标：** 确立全流程论证宪法——短片也需要主题论点来锚定创作方向。

**执行逻辑：**
1. 基于选定的核心戏剧动作，写一句话的主题论点（判断句，不是抽象命题如"关于母爱"）
2. 拆解正面立场和反面立场——反面立场必须有真实情感分量，不能是稻草人
3. > 📋 Read `core-methodology.md` →「一点五、主题论点」三步操作完整执行

**输出格式：**

* **主题论点：** [一句话判断句]
* **正面立场：** [故事要论证的答案]
* **反面立场：** [最强的反面论证，由谁/什么事件承载]

🔴 请回复"通过"进入下一步，或告诉我修改意见。

**[自检]**
- [ ] 主题论点是否是可争论的判断句（而非抽象名词短语）？
- [ ] 反面立场是否足够强——是否有人会真心认同它？
- [ ] 核心戏剧动作是否天然隐含了论点的正反冲突？

### 第二步：梗概草稿

**目标：** 让用户对整体故事的走向和气质有基本判断，再往下深入。

**执行逻辑：**
1. 基于用户选定的方案，写一段话（100-200字）的故事提要
2. 提要应包含：主角是谁、处于什么处境、因为什么事件被逼入什么困境、最终走向什么结局（可以模糊）
3. 提要的语气应该暗示全片的气质——是沉重还是轻快、是温暖还是冷冽

**输出格式：**

```
【梗概草稿】
（一段话的故事提要）
```

> 这个梗概在后续步骤中可以继续调整。请回复"通过"进入人物设计，或告诉我修改意见。

**[自检]**
- [ ] 梗概是否传达了故事的核心冲突？
- [ ] 读完梗概后，是否对"这是一个什么样的故事"有清晰感受？
- [ ] 梗概的语气是否暗示了全片的气质？

### 第三步：人物深度与弧光

**目标：** 拒绝纸片人，建立人物内在张力。

**执行逻辑：**
1. 设计人物外在需求 (Want) vs 内在欲望 (Need)
2. 设计人物小传——挖掘前史作为冲突根源
3. 设计人物矛盾性——外在表现 vs 内在真实
4. 除非用户特别要求，禁止过于夸张的设计
> 📋 Read `core-methodology.md` →「Want vs Need」→「人物四维——从抽象到具体的构建工具」→「三P生活——填充角色的外部世界」→「矛盾性」

**输出格式：**
* **人物：** [姓名]
* **外在需求：** [驱动动作的目标]
* **内在欲望：** [角色未意识到的灵魂缺口]
* **人物弧光：** [从A状态 → B状态]
* **人物小传关键点：** [关键前史]
* **矛盾特征：** [外在 vs 内在]

> 请回复"通过"进入下一步，或告诉我修改意见。你也可以让我[自检]。

**[自检：人物塑造与弧光]**
- [ ] 外在需求和内在欲望是否清晰且对立？
- [ ] 是否预设了不可逆的内在变化？
- [ ] 结局是否展示了"新常态"？
- [ ] 是否具备矛盾性？行为是否符合小传逻辑？

### 第四步：前史与世界观（短片精简版）

**目标：** 遵循"电影是现实生活的横截面"——故事开始之前，世界已经存在了。

短片不需要像长片那样完整建造世界观，但需要明确以下信息（编剧自己必须知道，不一定全部展示给观众）：

**输出格式：**
* **时代/环境背景：** [故事发生在什么时代、什么环境、什么社会氛围下]
* **角色前史关键事件：** [在故事开始之前，主角经历过什么导致了当前的处境和性格]
* **关系既往：** [如有重要关系，这段关系在故事开始之前是什么状态]
* **故事从哪个横截面切入：** [为什么选择从这个时刻开始讲——这个时刻有什么特殊性]

**注意：** 前史信息在剧本中通过对白和视觉线索自然散落，不要集中闪回或旁白交代。如非必要不用闪回。

### 第五步：结构大纲

**目标：** 搭建短片骨架，确保节奏递进。

**执行逻辑：**
1. 使用四段式结构
2. 只有一个单一核心事件，避免支线
3. 第二步中的重要前史细节需在第一幕和第二幕中自然展示（如非必要不用闪回）
4. 必须包含激励事件和"一切尽失"时刻
5. **开场的前15秒必须有视觉钩子**——第一个画面就应该包含信息或悬念，最有效的是一个"不太对"的日常画面
> 📋 Read `core-methodology.md` →「不归点——从建置到对抗的不可逆跨越」→「按类型的开场策略」→「故事的五级构成」检查价值翻转

**输出格式：**

* **第一幕：起因**
  * 开场钩子（前15秒的画面/动作）：
  * 建置（故事常态展现）：
  * 激励事件（打破平衡）：

* **第二幕：发展**
  * 冲突升级（障碍1）：
  * 冲突进一步升级（障碍2，不可省略）：
  * 中点（伪胜利/伪失败）和一切尽失：

* **第三幕：高潮**
  * 最终对决（动作顶点）：
  * 关键选择和故事反转：

* **第四幕：结局**
  * 新常态（弧光完成）：

### 第六步：场景拆解

**目标：** 将大纲转化为具体的视听单元。

**执行逻辑：**
1. 将第五步中每个情节点拆解为一个或几个核心戏剧动作（目标 → 障碍 → 结果）
2. 确保每个大的戏剧动作可拆解为几个微型戏剧动作
3. 每场戏标注预估时长和双轴叙事状态

**输出格式：**
* **第1场戏：** ...（约X秒）情节：松/中/紧 | 情感：轻/中/重
* **第2场戏：** ...（约X秒）情节：松/中/紧 | 情感：轻/中/重
* **...**
* **时长累计：** X分X秒 / 目标总时长X分钟

> 请回复"通过"进入下一步，或告诉我修改意见。你也可以让我[自检]。

**[自检]**
- [ ] 第五步中每个情节点是否都有具体拆分？

### 第七步：场景写作

**目标：** 填充血肉，执行视觉写作和潜台词。
**输入请求：** 请用户指定要生成的场景编号（或全部）。

**执行要求：**
1. **视觉写作**：严禁心理描写，只写能被看见的动作和能被听见的声音
2. **潜台词**：对话像冰山，避免直白。利用动作、停顿、回避表达真意
3. 标准剧本格式输出

## 短片节奏技法

### 前史展示
前史信息通过对白和视觉线索自然散落，不要集中闪回或旁白交代。

### 节奏的拉伸与回弹
- **拉伸**：用重复动作、平缓语速、静态画面建立预期
- **回弹**：在毫无预警处迅速打破平衡，用极短的视听语言制造节奏断裂

### 结局
结局应展示弧光完成后的"新常态"——角色变化后的世界是什么样的。不需要解释角色学到了什么，用一个画面让观众自己感受。
