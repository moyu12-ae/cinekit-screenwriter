# CineKit 编剧大师

> 本 Skill 基于 @山音 的「山音超级编剧大师」二次开发，新增 CineKit 持久化工作流与项目初始化系统。核心编剧方法论进行大量的补充，沿用与原版一致的 MIT 许可证。

---

## 这是什么？

CineKit 编剧大师是一个全格式影视编剧技能。你跟它对话，它引导你一步步完成剧本创作——从最初的故事概念，到人物设计、结构大纲、场景写作，最终产出一份完整的剧本。

**它能覆盖的格式：** 1-3分钟概念超短片 / 5-10分钟叙事短片 / 90分钟电影长片 / 多集剧集。

**它解决的核心问题：** 写剧本时容易犯的低级错误（心理描写、解释性台词、结构矛盾）会被系统性拦截。跨会话的创作进度会被持久化保存，关掉 Claude Code 也不会丢失。

## 快速开始（30秒上手）

### 第一次使用

在你的项目目录中，打开 Claude Code，说：

```
我要写剧本
```

CineKit 会自动初始化项目，然后问你：

> "你想做哪种体量的故事？1-3分钟概念片 / 5-10分钟短片 / 90分钟长片 / 多集剧集？"

选择格式后，它会引导你一步步完成八个创作步骤。**每一步完成后它会暂停等你确认**，不会自动跳到下一步。

### 继续上一次的创作

关掉 Claude Code 再打开，说：

```
继续写剧本
```

CineKit 会读取上次的进度，从你中断的步骤继续。

## 工作流全景

CineKit 遵循八步工作流，每步有明确的产出和暂停点：

| 步骤 | 做什么 | 产出文件 | 你能做什么 |
|:----:|--------|---------|-----------|
| **0** | 确认创作格式 | — | 选择概念片/短片/长片/剧集 |
| **1** | 确立故事核心概念 | `02_CONCEPT.md` | 讨论概念方向，选择一个最锐利的 |
| **1.5** | 确立主题论点 | `02_CONCEPT.md` | 确认故事想论证的"一句话真理" |
| **2** | 写梗概 + 填项目简报 | `02_CONCEPT.md` `01_BRIEF.md` | 确认故事提要，项目简报同步生成 |
| **3** | 设计人物 | `03_CHARACTERS.md` | 审查人物动机、矛盾、关系张力 |
| **4** | 建造世界观与前史 | `04_WORLD.md` | 确认世界规则和角色背景 |
| **5** | 搭建结构大纲 | `05_OUTLINE.md` | 审视结构、节奏、开场钩子 |
| **6** | 拆解场景 | `06_SCENES.md` | 确认每个场景的价值翻转（概念片可跳过） |
| **7** | 写全片剧本 | `07_SCRIPT.md` | 读剧本，提出修改意见 |
| **8** | 剧本诊断 | `08_DOCTOR.md` | 审查诊断报告，决定是否修改 |

**核心纪律：每一步完成后暂停等待你的指令。** 你需要回复「通过」才能进入下一步，也可以说「自检」让 CineKit 重新审视当前步骤，或直接提出修改意见。

## 四种格式怎么选？

| 格式 | 时长 | 适合讲什么 | 你需要提供什么 |
|------|------|-----------|--------------|
| **概念超短片** | 1-3分钟 | 一个高概念/反转，用视觉冲击力取胜 | 一个"如果……会怎样"的脑洞 |
| **短片** | 5-10分钟 | 一段完整的小故事，聚焦一个关键节点 | 一个你想讲的小故事梗概 |
| **长片** | 90分钟 | 完整的电影叙事，有充分空间展开人物深度 | 你对故事类型和大致方向的构想 |
| **剧集** | 多集 | 长篇叙事，可展开群像、暗线、多阶段发展 | 你对季度的整体构想 |

**拿不准选哪个？** 直接告诉 CineKit 你想讲什么故事，它会帮你判断最合适的格式。

## 一个完整的例子

以下是用户说"我想写个百合故事"后 CineKit 的实际工作流片段：

```
用户: 我要写剧本，百合故事

CineKit: 你想做哪种体量的故事？
         1-3分钟概念片 / 5-10分钟短片 / 90分钟长片 / 多集剧集？

用户: 1-3分钟概念片

CineKit: 🔴 路由确认：概念超短片，What-If型。确认无误？

用户: 确认。What-If型

CineKit: [生成3个概念方案：心光 / 彩色世界 / 反向引力]

用户: 概念B

CineKit: [展开主题论点 + 梗概]
         "暗恋最残酷的不是被拒绝，而是你永远不知道
          对方可能比你更早、更深地喜欢你。"

用户: 通过

CineKit: [设计A/B两个人物 + 世界观色彩规则]

... （逐步推进直到剧本完成）
```

每一步你都可以介入——修改概念、调整人物、重写某个段落。

## 什么是写作红线？

CineKit 在每次输出前都会内部自检以下红线。如果它自己想犯这些错误，会先拦截再输出：

- **不写心理描写**（"他意识到""她领会到"——摄影机拍不到）
- **不写括号暗示**（"（其实是在掩饰紧张）"——让动作自己说话）
- **不让角色解释设定**（"如你所知，我们是兄弟"——观众不傻）
- **不说教不煽情**（大段独白讲道理——用画面替代）
- **所有内容必须能被摄影机拍到**

## 项目文件说明

你的创作会被保存在项目目录下的 `.cinekit/` 文件夹中：

```
你的项目/
  .cinekit/
    PROGRESS.md            # 进度仪表盘——查看当前到哪一步了
    v1/
      01_BRIEF.md          # 项目简报——你的故事概览（Step 2 自动回填）
      02_CONCEPT.md        # 核心概念 + 主题论点 + 梗概
      03_CHARACTERS.md     # 人物设计
      04_WORLD.md          # 前史与世界观
      05_OUTLINE.md        # 结构大纲
      06_SCENES.md         # 场景拆解
      07_SCRIPT.md         # 全片剧本
      08_DOCTOR.md         # 剧本医生诊断报告
```

这些文件是 Markdown 格式，你可以用任何编辑器打开查看或修改。CineKit 会在下次会话时读取它们来恢复上下文。

## 诚实边界

CineKit **会**帮你做到：
- 消除所有菜鸟级写作错误（心理描写、解释性台词、结构矛盾）
- 建立严密的因果链和价值翻转
- 提供系统的结构模板和自检清单

CineKit **不能**帮你做到：
- 创造诺兰级别的核心概念（那需要天才的直觉，不是方法论）
- 写出让你自己都惊艳的"神级台词"（它能提示方向，但神来之笔来自你自己）

它在 L1（无硬伤）和 L2（结构严密）层面完全覆盖，在 L3（动人）层面只能提示方向。它让你成为一个合格的编剧，不是诺兰。
